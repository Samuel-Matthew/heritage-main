```

‚îî‚îÄ Good for: API consistency, error parsing

‚îú‚îÄ Why? Extract the important info
‚îú‚îÄ Extract: The error message
‚îî‚îÄ Good for: Handling all response types

‚îú‚îÄ Why? Users need friendly messages
‚îú‚îÄ Display: User-friendly toast
‚îî‚îÄ Good for: Great UX, happy users

Result:
‚úÖ Backend returns JSON (for API consistency)
‚úÖ Frontend parses JSON (for proper handling)
‚úÖ Users see Toast (for great UX)
‚úÖ No JSON shown to users!
```
